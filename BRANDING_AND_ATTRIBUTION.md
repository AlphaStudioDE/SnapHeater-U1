# Branding and Attribution Guidelines

This document explains how to refer to SnapHeater U1 in forks, remixes and derivative works.

It is not a separate software license. The source code remains licensed under the MIT License.

## Recommended wording for forks

If your project is based on SnapHeater U1, please use clear wording such as:

- "Based on SnapHeater U1 by Damian Borkowski"
- "Fork of SnapHeater U1"
- "Derived from the SnapHeater U1 firmware framework"

## Avoid confusion

Please do not present a modified fork as the original upstream SnapHeater U1 project. If your fork changes behavior, safety logic, hardware mapping or contest submission context, make that clear in your README.

## Project naming

You may refer to SnapHeater U1 for attribution. For heavily modified forks, using a distinct project name is recommended to avoid user confusion.

## Snapmaker name

"Snapmaker" is used descriptively to identify compatibility with Snapmaker U1. This project is not affiliated with or endorsed by Snapmaker unless explicitly stated by Snapmaker.
